## 트러블슈팅 (Troubleshooting)

### 1. Spotify API 호출 제한 문제
* **문제 상황**: 스포티파이 API에서 데이터를 가져올 때, 50개 이상의 데이터를 요청하면 에러가 발생하거나 데이터가 정확히 들어오지 않는 현상이 발생함.
* **원인 분석**: 스포티파이의 최근 API 정책 변경으로 인해 한 번에 요청 가능한 데이터 개수나 필드 정보가 제한됨.
* **해결 방법**: 
    - 무리하게 50개를 고집하지 않고, 안정적인 데이터 확보를 위해 `TOP_N` 변수를 10으로 조정함.
    - 단순히 데이터를 많이 가져오는 것보다, 구글 트렌드 데이터와 결합하여 '데이터의 질'을 높이는 방향으로 프로젝트 아키텍처를 재설계함.
* **배운 점**: API 사용 시 정책 변화에 유연하게 대응하는 법을 배웠으며, 에러 메시지를 단순히 '에러'로 보지 않고 '제한 조건'으로 재정의하는 사고를 하게 됨.# TrendScan
